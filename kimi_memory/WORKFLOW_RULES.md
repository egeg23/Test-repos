# 🐰 ПРАВИЛО РАБОТЫ: CodeRabbit + Kimi Workflow

## Обязательная схема для всех изменений кода

```
Помощник пишет код → PR → CodeRabbit проверяет → Я тестирую 
                                                           ↓
                                    Ты аппрувишь → Merge + обновление сервера
```

---

## 🔢 Шаги (строго по порядку)

### Шаг 1: Создание PR
**Кто:** Я (Kimi)  
**Как:** По запросу пользователя или помощника

**Команды для тебя:**
- `"@kimi создай PR с [описание изменений]"` — создаю ветку и PR
- Помощник может писать код сам → присылать мне → я создаю PR

**Результат:** PR на GitHub с номером (например, #5)

---

### Шаг 2: CodeRabbit проверяет (автоматически)
**Кто:** CodeRabbit (бот)  
**Время:** 1-3 минуты  
**Что проверяет:**
- Синтаксис Python
- Стиль кода (PEP 8)
- Безопасность (уязвимости, секреты)
- Логические ошибки
- Дублирование кода

**Результат:** Комментарии в PR с найденными проблемами

---

### Шаг 3: Я исправляю (при необходимости)
**Кто:** Я (Kimi)  
**Условие:** Если CodeRabbit нашёл критичные ошибки

**Что делаю:**
- Читаю комментарии CodeRabbit
- Перевожу на простой язык: "Было → Стало → Зачем"
- Показываю тебе кратко
- Получаю подтверждение: "Исправь"
- Вношу исправления в тот же PR
- CodeRabbit перепроверяет

---

### Шаг 4: Я тестирую на сервере
**Кто:** Я (Kimi)  
**Обязательно:** Да, всегда перед merge

**Что тестирую:**
- Импорты (нет ли ошибок импорта)
- Базовую функциональность
- Что ничего не сломалось
- Совместимость с текущим кодом

**Результат:** Отчёт в PR + рекомендация (Approve / Request changes)

---

### Шаг 5: Ты принимаешь решение
**Кто:** Ты (пользователь)  
**Варианты:**
- ✅ `"Смержи и обнови сервер"` — полное применение
- ✅ `"Смержи только"` — только GitHub, без сервера
- ⚠️ `"Поправь [что-то]"` — доработка
- ❌ `"Закрой PR"` — отмена

**Важно:** Я НЕ делаю merge без твоего явного разрешения!

---

### Шаг 6: Merge + обновление
**Кто:** Я (Kimi)  
**Действия:**
1. Нажимаю "Merge" на GitHub
2. Удаляю ветку
3. **Обновляю код на сервере** (если просил)
4. Проверяю, что всё работает
5. Отчитываюсь

---

## 📋 Быстрые команды

| Ты пишешь | Я делаю |
|-----------|---------|
| `"Создай PR для [задача]"` | Создаю ветку, коммичу, пушу, создаю PR |
| `"Проверь PR #N"` | Тестирую на сервере, пишу отчёт |
| `"Исправь PR #N"` | Вношу правки по комментариям CodeRabbit |
| `"Смержи и обнови сервер"` | Merge + git pull на сервере |
| `"Смержи только"` | Только merge на GitHub |
| `"Закрой PR #N"` | Закрываю без merge |

## ⚠️ КРИТИЧЕСКИЕ ПРАВИЛА

### Запрещено:
- ❌ Прямой push в `main` без PR
- ❌ Merge без тестов на сервере
- ❌ Игнорировать комментарии CodeRabbit о безопасности
- ❌ Обновлять сервер без явного подтверждения

### Обязательно:
- ✅ Всегда создавать PR для изменений
- ✅ Всегда ждать проверку CodeRabbit
- ✅ Всегда тестировать на сервере перед merge
- ✅ Получать твоё подтверждение перед merge
